# [!DNL Experience Manager] 릴리즈 차량 정의 업데이트  {#update-release-vehicle-definitions}

이 문서에는 [!DNL Adobe]이(가) 고객에게 제공하는 전체 릴리스, 기능 팩 및 서비스 팩 등 다양한 유형의 [!DNL Adobe Experience Manager] 릴리스에 대한 세부 정보가 포함되어 있습니다.

>[!NOTE]
>
>[!DNL Experience Manager] 업데이트 릴리스의 릴리스 일정은 [[!DNL Experience Manager] 업데이트 릴리스 로드맵](update-releases-roadmap.md)을 참조하십시오.

## 전체 릴리스 {#full-release}

| 항목 | 설명 |
|-------|------|
| 정의 | <ul> <li> 예약된 릴리스 </li> <li> 릴리스 노트에 정의된 특정 버전에 대한 업그레이드 경로 지원 </li> </ul> |
| 이름 지정 | <ul> <li> 주요 릴리스의 버전 번호는 X+1.Y.Z 공식에 따라 증가합니다. </li> <li> 부 릴리스의 버전 번호는 공식 X.Y+1.Z를 기준으로 증가합니다 </li> </ul> X는 기본 버전 번호이고 Y는 보조 버전 번호이고 Z는 패치 번호입니다. |
| 포함 | <ul> <li> 새로운 기능 </li> <li>  향상된 기능 </li> <li>  버그 수정 </li> </ul> |
| 설명서 | <ul> <li> 릴리스 노트는 설명서 포털에서 확인할 수 있습니다. </li> <li> 기능, 개선 사항 및 버그 수정에 대한 설명서는 설명서 포털에서 확인할 수 있습니다 </li> </ul> |
| Cadence | 연간 |
| 가용성 및 설치 | <ul> <li> 독립 실행형 제품 설치 프로그램으로 제공 </li> <li>  라이센스 웹 사이트 및 Managed Services을 통해 제공 </li> <li> 라이센스 웹 사이트 컨텐츠 저장소 마이그레이션이 필요할 수 있음 </li> </ul> |
| 테스트 수준 | QA에 의해 완전히 검증됨 |

## 서비스 팩 {#service-pack}

| 항목 | 설명 |
|-----|-----|
| 정의 | <ul> <li> 예약된 릴리스 </li> <li> 현재 롤백할 수 없습니다. </li> </ul> |
| 이름 지정 | <ul> <li> 패치 릴리스 번호는 한 자리 숫자입니다. </li> <li> 설치 후 X.Y.Z.SPx 공식을 기준으로 설치된 릴리스 번호 패치 숫자가 증가합니다 </li> </ul> X는 기본 버전 번호이고 Y는 보조 버전 번호이고 Z는 패치 번호입니다. x는 서비스 팩 번호입니다. |
| 포함 | <ul> <li> 새로운 기능</li> <li>  향상된 기능 </li> <li> 버그 수정 </li> <li> 공통 관심 기능 팩(있는 경우) </li> </ul> |
| 설명서 | <ul> <li> 설명서 포털에 있는 릴리스 노트 </li> <li> 설명서 포털의 기능, 개선 사항, 버그 수정 사항 </li> </ul> |
| Cadence | 분기별 |
| 가용성 및 설치 | <ul> <li> 패키지로 제공 </li> <li> 소프트웨어 배포 시 사용 가능</li> <li>  기존 기능 설치 필요 </li> </ul> |
| 테스트 수준 | <ul> <li> 모든 수정 사항 QA 유효성 검사 </li> <li>  자동화를 통한 전체 패키지 상태 유지 </li> </ul> |

## 누적 수정 팩  {#cumulative-fix-pack-aem}

| 항목 | 설명 |
|-----|-----|
| 정의 | <ul> <li> 수정 사항 릴리스의 단일 전달 모델 </li> <li> 개별 구성 요소의 컨텐츠 패키지를 포함하는 수집기 컨텐츠 패키지 </li> <li>  CFP는 핫픽스를 롤오버하고 있으며 그 일부에는 향상된 기능이 없습니다.  </li> </ul> |
| 이름 지정 | X.Y.Z.CFPx <br> X는 기본 버전 번호이고, Y는 보조 버전 번호이고, Z는 패치 번호입니다. x는 누적 서비스 팩 번호입니다. |
| 포함 | CFP는 지정된 날짜까지 모든 구성 요소의 수정 사항이 포함된 누적 수정 팩입니다. 예를 들어 고객이 CFP3을 적용하면 CFP3 = CFP1 + CFP2가 됩니다. |
| 설명서 | 설명서 포털에 있는 릴리스 노트 |
| Cadence | 분기별 |
| 가용성 및 설치 | <ul> <li> 패키지로 제공 </li> <li>  소프트웨어 배포 시 사용 가능 </li> <li>  최신 서비스 팩에 따라 다름 </li> <li>  CFP는 자체 의존적입니다. 고객은 종속성을 찾거나 해결할 필요가 없습니다. CFP는 최신 릴리스 서비스 팩에 설치해야 합니다. </li> <li>  CFP는 단일 패키지로 설치하여 고객 경험을 향상시킬 수 있습니다.  </li> </ul> |
| 테스트 수준 | 통합 수준 및 회귀 테스트에서 QA가 확인됨 |

## 오버레이 {#overlay}

| 항목 | 세부 사항 |
|-------|--------|
| 이름 지정 | overlay-&lt;ticket ID> |
| 포함 | JS 또는 JSP 파일에 대한 버그 수정 |
| 설명서 | 없음 |
| Cadence | 필요한 경우 |
| 가용성 및 설치 | <ul> <li> [!DNL Experience Manager] 고객 지원 센터에서 패키지로 제공  </li> <li> 서비스 팩이나 전체 릴리스에 반드시 포함되지 않음 </li> </ul> |
| 테스트 수준 | 고객 지원 센터에서 확인 |

## 기능 팩 {#feature-pack}

| 항목 | 세부 사항 |
|--------|-----|
| 정의 | <ul> <li>기능 팩은 추가 기능이며 서비스 팩을 통해 제공됩니다. [!DNL Experience Manager] 버전이 마지막 서비스 팩을 릴리스한 경우 Adobe은 향후 해당 기능에 대한 기능 팩을 제공하지 않습니다. </li> <li> FP에는 후속 제품 릴리스로 예정되어 있는 제품 개선 사항이 포함되지만, [!DNL Adobe's] 제품 관리의 결정에 따라 일찍 제공됩니다.</li> <li>  기능은 항상 다음 주요 릴리스와 병합된 후 고객이 필요로 하는 [!DNL Experience Manager] 버전으로 포팅됩니다. </li> <li>  공통 관심 및 GA 기능 팩은 다음 서비스 팩에 병합됩니다.  </li> </ul> |
| 이름 지정 | `cq-<Release Version>-featurepack-<feature pack ID>-<feature pack version>` |
| 포함 | <ul> <li> 새로운 기능 </li> <li> 향상된 기능 </li> <li> 버그 수정(증분 제품 업데이트) </li> </ul> |
| 설명서 | 설명서는 adobe.com에서 제공됩니다. |
| Cadence | 제품 영역에 따라 다름 |
| 가용성 및 설치 | <ul> <li>서비스 팩을 통해 제공 </li> <li> 소프트웨어 배포에 있음. 고객은 소프트웨어 배포를 통해 [!DNL Adobe's] 약관에 동의합니다. </li> </ul> |
| 테스트 수준 | 일반 가용성 기능 팩은 QA의 유효성을 검사합니다. |

* 1:OAK 수정 사항은 개별 핫픽스로 제공되지 않습니다. 하지만 다음의 누적 Oak 핫픽스에 포함됩니다. 필요한 경우 최신 COFP 위에 진단 빌드를 제공할 수 있습니다. 전제 조건은 고객에게 최신 COFP가 실행된다는 것입니다. 진단 빌드는 핫픽스와 동일한 수준의 품질 보증만 제공합니다. 따라서 누적 수정 팩, 서비스 팩 또는 제품 릴리스와 동일한 수준의 품질 보증을 제공하지 않습니다. 최종 수정 내용은 다음 CFP와 함께 제공됩니다.
